1.类别
    静态代理：每一个代理类都必须实现一遍委托类（也就是realsubject）的接口，如果接口增加方法，则代理类也必须跟着修改。
        其次，代理类每一个接口对象对应一个委托对象，如果委托对象非常多，则静态代理类就非常臃肿，难以胜任。
    动态代理：
        重要的接口
            InvocationHandler ：接口是给动态代理类实现的，负责处理被代理对象的操作的
        重要的类
            Proxy ：用来创建动态代理类实例对象的，因为只有得到了这个对象我们才能调用那些需要代理的方法。

        使用 Proxy.newProxyInstance（）生成的代理对象Proxy去调用具体实现类的方法，系统会将此方法分发给invoke()。
